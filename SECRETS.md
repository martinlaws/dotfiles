# Secrets restore checklist

Secrets are **not** in this repo. They live in **1Password** and, at runtime, in
per-project `.env.local` files (gitignored) — there is no single global
`secrets.env`. This file is the checklist for putting them back on a new machine.

## How secrets are laid out

- **Per-project**: each repo under `~/code` that needs keys has its own
  `.env.local` (e.g. `chaos/dashboard/.env.local`). These are gitignored and
  never committed. Recreate them from 1Password per project.
- **1Password is the source of truth.** If a key isn't there, add it there
  first, then copy into the project that needs it.

## Keys to restore (names + where to get them)

| Key | Used by | Where to get it |
|---|---|---|
| `ANTHROPIC_API_KEY` | Claude API scripts, tooling | console.anthropic.com → API Keys |
| `OPENAI_API_KEY` | misc AI scripts | platform.openai.com → API Keys |
| `PERPLEXITY_API_KEY` | deep-research / Perplexity runs | perplexity.ai → Settings → API |
| `CALCOM_API_KEY` | /avails + /book (chaos dashboard) | Cal.com → Settings → Developer → API Keys. Lives in `chaos/dashboard/.env.local` |
| `GITHUB_TOKEN` | gh / CI scripts (if not using gh auth) | github.com → Settings → Developer settings → PAT |

> Not exhaustive — check each project's `.env.local.example` (if present) or
> `git grep -i 'process.env\.' ` in the project for the authoritative list.

## Other credentials (not env vars)

- **GitHub SSH key** — generated by `scripts/setup-ssh.sh`; add the public key
  to github.com/settings/keys. Required before cloning private repos
  (including `claude-config`).
- **Claude Code auth** — re-run Claude Code and sign in; it regenerates
  `~/.claude/.credentials.json`. Not restored from any repo.
- **1Password itself** — sign in first thing; everything else depends on it.
